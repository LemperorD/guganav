# B-spline 轨迹优化模块 — 设计文档

## 1. 整体流程

```
JPS路径 (稀疏跳转点, N个航点)
  │
  ├──→ fit()           ←  B样条拟合: 插值→控制点提取→降采样
  │
  ├──→ optimize()      ←  梯度下降优化: 代价函数 → 控制点移动 → 障碍物投影
  │
  └──→ sample()        ←  均匀参数采样, 输出M个平滑航点
```

## 2. 数学背景

### 2.1 B样条基函数与曲线

B样条曲线由三要素定义：
- **阶数 k** (degree), k = 7 (固定)
- **控制点** $P_0, ..., P_{M-1}$, M 个二维点
- **节点向量** $\mathbf{t} = [t_0, ..., t_{M+k}]$ (长度 = M + k + 1, 通常取 t=`KnotAveraging`)

**B样条求值公式** (Cox-de Boor 递推):

$$C(u) = \sum_{i=0}^{M-1} N_{i,k}(u) \cdot P_i$$

其中 $N_{i,k}(u)$ 是 k 阶 B 样条基函数，在参数区间 $u\in[t_k, t_M]$ 上递推求值。

### 2.2 曲率计算

曲线 $C(u) = (x(u), y(u))$ 在参数 u 处的曲率 (κ):

$$\kappa(u) = \frac{|x'(u) y''(u) - y'(u) x''(u)|}{(x'(u)^2 + y'(u)^2)^{3/2}}$$

利用 `Eigen::Spline::derivatives<2>(u)` 一次求出一阶和二阶导数。

### 2.3 控制点数量与拟合精度

从 JPS 路径 (N 个航点) 生成 M 个 B 样条控制点的过程:

1. **完整插值**: `SplineFitting::Interpolate(pts, 7)` → 产生 N 个控制点, **这个中间样条精确穿过所有 JPS 航点**
2. **降采样**: 从插值样条上以均匀参数间隔采样 M 个控制点

**降采样的代价**: M < N 时, 降采样控制点构造的 B 样条不再精确穿过原始航点, 只能近似。

## 3. 控制点选取策略

### 3.1 当前实现 (有问题!)

```cpp
// fit() 中的降采样:
int M = min(max_control_points, n_pts);  // default: M <= 15
for (int i = 0; i < M; ++i) {
    double u = (double)i / (double)(M - 1);  // 均匀参数!
    Eigen::Vector2d p = dense_spline(u);     // 从 dense 插值样条采样
    state_.control_points(0, i) = p.x();
    state_.control_points(1, i) = p.y();
}
```

**问题 1**: 均匀参数采样 (u = i/(M-1)) 忽略了原始 JPS 航点的 chord-length 分布。JPS 跳转点在弧长上不均匀分布 (近跳转点密集, 远跳转点稀疏)。均匀采样会导致控制点与实际航点位置错位。

**问题 2**: 因为降采样后的控制点不再是原始插值控制点, 重建的 B 样条**不再穿过原始航点**。即使 distance_weight=500, 梯度下降也无法补偿信息损失。

### 3.2 正确的策略: 保留全部插值控制点

B 样条插值 ($N$ 个航点 → $N$ 个控制点) 已经产生了一条精确穿过所有 JPS 航点的样条曲线。**不需要**降采样控制点。

**关键洞察**: 优化应该在**全精度**上进行 — 遍历所有控制点 (M=N), 但只修改**偏离过大**的控制点。平滑性代价通过**最小化 2 阶导数**来控制, 而不是通过减少控制点数量。

结论: `max_control_points` 参数应设为非常大的值 (如 200 或 INF), 改为在 `fit()` 中不使用降采样:

```cpp
// 直接使用插值样条的控制点, 不降采样:
const auto & ctrl = fitted_spline.ctrls();
state_.control_points = ctrl;
state_.knots = knot_vec;  // 使用原始 KnotAveraging 的节点向量
```

## 4. 代价函数设计

### 4.1 三项加权代价

$$J(\mathbf{P}) = w_s J_{smooth}(\mathbf{P}) + w_d J_{dist}(\mathbf{P}) + w_o J_{obs}(\mathbf{P})$$

### 4.2 平滑性代价 $J_{smooth}$

对曲线的 2 阶导数 (曲率能量) 做数值积分:

$$J_{smooth}(\mathbf{P}) = \frac{1}{K+1}\sum_{i=0}^{K} \|C''(u_i)\|^2$$

其中 $u_i = i/K$, K = 50 (在 evalCost 中)。2 阶导数通过 `spline.derivatives<2>(u)` 获取。

**物理意义**: 2 阶导数是法向加速度, 与路径曲率成正比。最小化它→路径更平直, 拐弯更舒缓。

### 4.3 距离保持代价 $J_{dist}$

对每个原始 JPS 航点 $q_i$ 及其 chord-length 参数 $\tau_i$:

$$J_{dist}(\mathbf{P}) = \sum_{i=0}^{N-1} w_d \cdot \|C(\tau_i) - q_i\|^2$$

**关键**: 这里评估的是 **B 样条在原始航点参数处的值** 与 **原始航点** 的距离。如果控制点没有偏离, C(τ_i) 应该恰好等于 q_i (因为样条是通过插值产生的)。

### 4.4 障碍物代价 $J_{obs}$

对样条做密集采样 (K=200), 检查每个采样点是否落入代价图中 cost≥253 的障碍物格子:

$$J_{obs}(\mathbf{P}) = \sum_{i: C(u_i) \in \text{obstacle}} w_o \cdot (1 + |\Delta x| + |\Delta y|)$$

其中 Δx, Δy 是采样点相对于障碍物格子中心的偏移量。

## 5. 优化器: 基于梯度的下降 + 回溯线搜索

### 5.1 算法

```
输入: 初始控制点 params, 代价函数 f
参数: 初始步长 α=1e-3, 数值差分步长 h=1e-3, 梯度容差 1e-8

while iter < max_iters:
    # 1. 数值梯度 (中心差分)
    for each param i:
        grad[i] = (f(params[i]+h) - f(params[i]-h)) / (2h)

    # 2. 收敛检查
    if ||grad|| < gtol: 退出

    # 3. 回溯线搜索
    for step in 1..15:
        params_try = params - α * grad
        if f(params_try) < f_best:
            params = params_try; α *= 2; break
        else:
            α *= 0.5
```

### 5.2 优化变量

- **自由变量**: M-2 个内部控制点 (2 个维度每个) → 2(M-2) 维
- **固定变量**: 首尾两个控制点 (起点和终点不变)

## 6. 障碍物避让: 控制点投射

优化后对每个控制点做后处理:

```
for each interior ctrl pt (x, y):
    if costmap[int(x)][int(y)] >= 253:
        在半径 R=1..10 的方形区域内搜索最近的 <253 格子
        将控制点移动到该格子的中心
        找到后立即跳出
```

## 7. 当前存在的问题汇总

### 7.1 [根本问题] 控制点降采样导致路径偏离

`fit()` 在从 N 个插值控制点降采样到 M 个均匀参数控制点时, 丢弃了大部分信息。重建的 B 样条不再精确穿越原始 JPS 航点, 导致 distance_cost 无法驱动优化回到正确轨道。

### 7.2 [次要问题] 距离代价权重不够大

即使 distance_weight=500, 平滑性代价仍主导优化方向。在障碍物拐角处, 平滑性的梯度 (减少曲率) 大于距离约束的梯度 (回归航点), 导致曲线偏离。

### 7.3 [次要问题] 数值梯度过大

h=1e-3 用于代价图采样 (K=200) 时, 障碍物代价对控制点位置的变化不敏感 — 移动 1e-3 单元不会改变 `int(x)` 的值, 因此障碍物代价梯度为零, 优化器'看不见'障碍物。

### 7.4 [次要问题] 参数耦合强度不够

优化变量 2(M-2) 维, 但只有 N 个原始航点 (通常 N≈M), 约束不足。理想情况下应该是约束数量 >> 自由变量数量 (N >> M)。

## 8. 修复方案

### 8.1 去掉控制点降采样, 保持全部插值控制点

`fit()` 直接使用 SplineFitting 产生的所有 N 个控制点, 不降采样。

### 8.2 减小优化自由度: 只允许控制点移动有限距离

使用约束条件: 每个控制点只能在以其初始位置为中心的方框内移动 (如±2 单元)。这可以通过在 `gradientDescent` 的 `x_try` 步后做 `clamp` 实现:

```cpp
for (int i = 0; i < N; ++i) {
    x_try[i] = std::clamp(x_try[i], init_value - corridor, init_value + corridor);
}
```

`corridor` 默认取 1.5~2.5 单元 (即控制点只能在原位置附近微小调整)。

### 8.3 调整默认权重

- smoothness_weight: 0.1 (降低, 不再主导)
- distance_weight: 10.0 (降低, 因为控制点已经精确插值)
- obstacle_weight: 50000 (保持, 确保避障)

### 8.4 增大障碍物代价的数值梯度步长

将 h 从 1e-3 增大到 0.5 (半个格子的宽度), 这样数值梯度才能'感知'到障碍物边界的移动。

### 8.5 简化: 不做梯度下降优化 (初始插值已经最优)

对于 B 样条插值, 初始曲线已经满足所有约束 (精确穿越航点, 曲率由 B 样条内在保证)。**不需要梯度下降优化路点位置**。
如果存在障碍物冲突, 只需要在 `sample()` 后做路径点的障碍物投射 (类似控制点投射), 即可确保输出路径无碰撞。

### 8.6 最终推荐方案

1. `fit()` 使用全量插值控制点 (不降采样)
2. `optimize()` 跳过梯度下降 (初始插值已最优)
3. `optimize()` 仅做控制点障碍物投射 (现有的 post-projection 逻辑)
4. `sample()` 后对采样点做障碍物检查, 修正碰撞点
5. 保留梯度下降代码作为可选项 (可通过 config 开关), 但默认不启用
